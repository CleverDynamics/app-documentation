    Bespoke Examples

    DataItem ‘NoOfPackages’

    This bespoke Codeunit subscribes to events in Handheld for Warehousing – specifically codeunit HHW Std. Trans. Activity Mgt. - adding a “No. Of Packs” DataItem to warehouse activities after Initialise, and populating a field on Warehouse Activity Line immediately before registering the activity…

    codeunit 50100 HHBespokeFunctions

    {

    var

    SessionMgt: Codeunit "HHF Session Mgt.";

    [EventSubscriber(ObjectType::Codeunit, Codeunit::"HHW Std. Trans. Activity Mgt.",

    'OnAfterInitActivity', '', false, false)]

    procedure OnAfterInitReceipt(var Sender: codeunit "HHW Std. Trans. Activity Mgt.")

    begin

    SessionMgt.InitResponseData('NoOfPacks', 'No. Of Packs', false);

    end;

    [EventSubscriber(ObjectType::Codeunit, Codeunit::"HHW Std. Trans. Activity Mgt.",

    'OnBeforePostActivity', '', false, false)]

    procedure OnBeforeRegisterActivity(var Sender: codeunit "HHW Std. Trans. Activity Mgt.";

    var WhseActLine : record "Warehouse Activity Line")

    begin

    WhseActLine."Shipment Units" := SessionMgt.GetContent('NoOfPacks');

    end;

    }

    Note that SessionMgt is used for handling DataItems, not .Sender.

    The dependencies section in app.json will require the following entries...

    "dependencies": [

    {

    "appId": "1e6af11a-9854-46d4-a4b2-3ae0cf4d7fad",

    "name": "Handheld Framework",

    "publisher": "Clever Dynamics",

    "version": "7.1.0.0"

    },

    {

    "appId": "b9207dc8-e910-4c0d-85fc-7b7ec3e03112",

    "name": "Handheld for Warehousing",

    "publisher": "Clever Dynamics",

    "version": "7.1.0.0"

    }

    Picking Example: Validate ‘ActivityNo’

    This bespoke codeunit subscribes to the events and uses the ValidateTransDataItem function to reduce the overhead of code.

    On Initialise, ActivityNo is set to Blank and is not Locked - It will be prompted for.

    After Initialise a Lookup is built for Warehouse Shipments which have Warehouse Activity Pick Lines - ‘ShipmentKey’.

    When this is validated ‘ActivityNo’ is Validated with the first pick line assigned to the chosen Warehouse shipment.

    This is advantageous because now all the standard Handheld for Warehousing validation code for ‘ActivityNo’ will take place with the developer having to replicate anything or write any more code. The new value will be validated into ‘ActivityNo’, the user will not have to prompt for it and all the dependant fields will be populated accordingly.

    The flow is as follows:

    codeunit 50100 HHBespokeFunctions

    {

    var

    SessionMgt: Codeunit "HHF Session Mgt.";

    [EventSubscriber(ObjectType::Codeunit, Codeunit::"HHW Std. Trans. Activity Mgt.",

    'OnAfterInitActivity', '', false, false)]

    procedure OnAfterInitReceipt(var Sender: codeunit "HHW Std. Trans. Activity Mgt.")

    var

    WarehouseShipmentHeader: record "Warehouse Shipment Header";

    tmpWarehouseShipHeader: record "Warehouse Shipment Header" temporary;

    WarehouseActivityLine: record "Warehouse Activity Line";

    RecRef: RecordRef;

    ColumnStyle: record "HHF Handheld Content";

    begin

    with WarehouseShipmentHeader do begin

    //Insert as first DataItem

    SessionMgt.InsertTransResponseData('ShipmentKey', 'ZoneKey', 'ShipmentKey', false);

    SetRange("Location Code", SessionMgt.GetLocationCode());

    tmpWarehouseShipHeader.Reset();

    tmpWarehouseShipHeader.DeleteAll();

    repeat

    //Only Shipments which have Pick Lines

    WarehouseActivityLine.SetRange("Whse. Document No.", "No.");

    WarehouseActivityLine.SetRange("Whse. Document Type",

    WarehouseActivityLine."Whse. Document Type"::Shipment);

    if not WarehouseActivityLine.IsEmpty() then begin

    tmpWarehouseShipHeader.Copy(WarehouseShipmentHeader);

    tmpWarehouseShipHeader.Insert();

    end;

    until Next() = 0;

    //Build lookup

    RecRef.GetTable(tmpWarehouseShipHeader);

    SessionMgt.AddTransLookupColumnStyle('ShipmentKey', 'DocumentNo', FieldCaption("No."),

    Database::"Warehouse Shipment Header", FieldNo("No."), ColumnStyle.Importance::Promoted);

    SessionMgt.AddTransLookupColumnStyle('ShipmentKey', 'PostingDate', FieldCaption("Posting Date"),

    Database::"Warehouse Shipment Header", FieldNo("Posting Date"),

    ColumnStyle.Importance::Promoted);

    SessionMgt.BuildTransLookup('ShipmentKey', RecRef);

    end;

    end;

    [EventSubscriber(ObjectType::Codeunit, Codeunit::"HHW Std. Trans. Activity Mgt.",

    'OnAfterValidateActivity', '', false, false)]

    procedure OnAfterValidateActivity(var Sender: codeunit "HHW Std. Trans. Activity Mgt.")

    var

    WarehouseShipmentHeader: record "Warehouse Shipment Header";

    WarehouseActivityLine: record "Warehouse Activity Line";

    CurrDataItem: Text;

    begin

    with WarehouseShipmentHeader do begin

    CurrDataItem := SessionMgt.GetTransCurrentDataItem();

    case CurrDataItem of

    'ShipmentKey':

    begin

    //Find warehouse shipment

    SetPosition(SessionMgt.GetContent('ShipmentKey'));

    SetRecFilter();

    FindFirst();

    //Find activity line

    WarehouseActivityLine.SetRange("Activity Type", WarehouseActivityLine."Activity Type"::Pick);

    WarehouseActivityLine.SetRange("Whse. Document No.", WarehouseShipmentHeader."No.");

    WarehouseActivityLine.SetRange("Whse. Document Type",

    WarehouseActivityLine."Whse. Document Type"::Shipment);

    WarehouseActivityLine.FindFirst();

    SessionMgt.SetTransResponseData('ZoneCode', 'PICK', true);

    SessionMgt.SetTransResponseData('ZoneKey', '', true);

    //Validate 'ActivityNo'

    SessionMgt.ValidateTransDataItem('ActivityNo', WarehouseActivityLine."No.", true);

    end;

    end;

    end;

    end;

    }

    Function - Consolidate

    This is a real example of a function originally created for one of our customers under a previous version of Handheld: A bespoke pallet/container confirmation screen was required. This function uses the bespoke tables “Container Header” and “Container Line” and sets a Boolean confirmation that the container has been consolidated.

    A function was added manually to the Device Function table and added to the device menu.

    A new codeunit called “Custom Handheld Functions” was created.

    Four new handlers were created that hooked into OnFunctionInitialise, OnFunctionValidate, OnFunctionCancel and OnFunctionPost respectively.

    [EventSubscriber(ObjectType::Codeunit, Codeunit::"HHF Web Service Trans.", 'OnFunctionInitialise', '', false, false)]

    procedure OnFunctionInitialise(FunctionCode: Code[20])

    begin

    CASE FunctionCode OF

    'CONSOLIDATE':

    InitConsolidate;

    END;

    end;

    [EventSubscriber(ObjectType::Codeunit, Codeunit::"HHF Web Service Trans.", 'OnFunctionValidate', '', false, false)]

    procedure OnFunctionValidate(FunctionCode: Code[20]; VAR ValidationItems: Record "HHF Handheld Content")

    begin

    CASE FunctionCode OF

    'CONSOLIDATE':

    ValidateConsolidate(ValidationItems);

    END;

    end;

    [EventSubscriber(ObjectType::Codeunit, Codeunit::"HHF Web Service Trans.", 'OnFunctionCancel', '', false, false)]

    procedure OnFunctionCancel(FunctionCode: Code[20]; VAR ValidationItems: Record "HHF Handheld Content")

    begin

    CASE FunctionCode OF

    'CONSOLIDATE':

    CancelConsolidate(ValidationItems);

    END;

    end;

    [EventSubscriber(ObjectType::Codeunit, Codeunit::"HHF Web Service Trans.", 'OnFunctionPost', '', false, false)]

    procedure OnFunctionPost(FunctionCode: Code[20])

    begin

    CASE FunctionCode OF

    'CONSOLIDATE':

    PostConsolidate;

    END;

    end;

    The codeunit is designed to cater to multiple bespoke functions, hence the use of CASE statements.

    Functions were then created for each consolidation request type.

    procedure InitConsolidate()

    begin

    //Initialise---

    SessionMgt.InitResponseData('ContainerNo', ContainerCaption, FALSE);

    SessionMgt.InitResponseData('ShipmentNo',

    ContainerHeader.FIELDCAPTION("Warehouse Shipment No."), TRUE);

    SessionMgt.InitResponseData('Status', StatusCaption, TRUE);

    end;

    Three DataItems are initialised: ContainerNo, ShipmentNo and Status.

    procedure ValidateConsolidate(VAR ValidationItems: Record "HHF Handheld Content")

    var

    WhseActLine: record "Warehouse Activity Line";

    begin

    //Validate---

    CASE ValidationItems."Column Name" OF

    'ContainerNo':

    BEGIN

    ContainerHeader.GET(ValidationItems.Text);

    //Check there are no in-progress pick activities---

    WhseActLine.RESET;

    WhseActLine.SETCURRENTKEY("Whse. Document No.", "Whse. Document Type");

    WhseActLine.SETRANGE("Whse. Document No.", ContainerHeader."Warehouse Shipment No.");

    WhseActLine.SETRANGE("Whse. Document Type", WhseActLine."Whse. Document Type"::Shipment);

    IF WhseActLine.FIND('-') THEN

    REPEAT

    IF NOT WhseDocAddl.GET(DATABASE::"Warehouse Activity Line",

    WhseActLine."Activity Type", WhseActLine."No.", WhseActLine."Line No.") THEN

    ERROR(CannotFindErr, ContainerHeader.Code, ContainerHeader."Warehouse Shipment No.");

    IF NOT WhseDocAddl.Suspended THEN

    ERROR(CannotFindErr, ContainerHeader.Code, ContainerHeader."Warehouse Shipment No.");

    UNTIL WhseActLine.NEXT = 0;

    ContainerHeader.CALCFIELDS("Not consolidated");

    //Set response data---

    IF ContainerHeader."Not consolidated" THEN

    SessionMgt.SetTransResponseData('Status', StatusNotConfirmed, TRUE)

    ELSE

    SessionMgt.SetTransResponseData('Status', StatusAlreadyConfirmed, TRUE);

    SessionMgt.SetTransResponseData('ContainerNo', ContainerHeader.Code,

    ContainerHeader."Not consolidated");

    SessionMgt.SetTransResponseData('ShipmentNo', ContainerHeader."Warehouse Shipment No.", TRUE);

    //Set column style (Visual elements)---

    SessionMgt.AddTransColumnStyleGroup('ContainerNo', ContainerCaption);

    SessionMgt.AddTransColumnStyle('ContainerNo', 'ContainerNo', ContainerCaption, 200);

    SessionMgt.AddTransColumnStyle('ContainerNo', 'ShipmentNo',

    ContainerHeader.FIELDCAPTION("Warehouse Shipment No."), 200);

    SessionMgt.AddTransColumnStyle('ContainerNo', 'Status', StatusCaption, 200);

    END;

    END;

    end;

    The ValidationItems are cycled through in turn (We only need to worry about validating the container). We check the data is valid and set the responses accordingly (SetResponseData). We also add visual elements or ColumnStyles (AddColumnStyleGroup and AddColumnStyle).

    procedure CancelConsolidate(VAR ValidationItems: Record "HHF Handheld Content")

    begin

    //Cancel---

    CASE ValidationItems."Column Name" OF

    'ContainerNo':

    InitConsolidate;

    END;

    end;

    Cancellation simply calls InitConsolidate to re-initialise the DataItems.

    Finally, The Post function…

    procedure PostConsolidate()

    begin

    //Post---

    ContainerLine.RESET;

    ContainerLine.SETRANGE("Container No.", SessionMgt.GetContent('ContainerNo'));

    ContainerLine.MODIFYALL(Consolidated, TRUE, TRUE);

    SessionMgt.SetTransResponseData('ContainerNo', SessionMgt.GetContent('ContainerNo'), FALSE);

    SessionMgt.SetTransResponseData('Status', StatusConfirmed, TRUE)

    end;

    Sets Consolidated on the container lines and marks the Status DataItem as ‘Confirmed’.

    Initialise > Validate > Post

    It’s also worth adding an event subscriber for table HHF Device Function event OnAfterGenerateFunctions so that the bespoke function is re-added when “Generate Functions” is clicked.

    "name": "Handheld Framework",

    "publisher": "Clever Dynamics",

    "version": "7.1.0.0"

    }

    ],
